# 桌面学习助手 - 需求分析文档

## 一、项目概述

### 1.1 项目背景

**项目名称**: 桌面学习助手

**产品定位**: 面向学生群体的桌面端学习辅助工具，帮助用户通过视频和文档两种渠道进行高效学习，自动生成和管理课堂笔记。

**目标用户**:
- 大学生、高中生等需要记录课堂笔记的学生群体
- 需要复习课程录像的在线学习者
- 需要从文档中提取知识点的自学者

### 1.2 用户痛点分析

| 痛点 | 描述 | 影响 |
|------|------|------|
| **课堂笔记记录困难** | 上课时需要同时听讲、思考、记录，容易遗漏重点内容 | 学习效率降低，复习时资料不完整 |
| **视频学习效率低** | 观看教学视频时需要反复拖动进度条查找关键信息 | 时间浪费，学习体验差 |
| **文档信息提取耗时** | 从PDF/DOCX等文档中手动整理知识点、标记要点非常耗时 | 学习动力下降，容易放弃 |
| **知识结构不清晰** | 缺乏系统化的知识组织方式，笔记散乱 | 复习效果差，知识体系不完整 |
| **疑难点记录不完善** | 遇到疑难问题时的理解和解释容易被遗忘 | 后期复习时无法回溯理解过程 |

---

## 二、功能需求分析

### 2.1 功能架构图

```
桌面学习助手
├── 视频学习模块
│   ├── 语音识别转文字
│   ├── 一键截图保存
│   └── 图像内容识别
├── 文档学习模块
│   ├── 文档导入与解析
│   ├── 知识目录生成
│   ├── 要点标记
│   └── 疑难点标注
└── 课堂笔记模块
    ├── 笔记创建与管理
    ├── 内容编辑与整理
    └── 笔记导出与分享
```

### 2.2 详细功能规格

#### 2.2.1 视频学习模块

##### 功能1: 语音识别转文字

| 属性 | 说明 |
|------|------|
| **描述** | 实时识别视频中的语音内容，转换为文字并自动保存到课堂笔记中 |
| **优先级** | P0 (核心功能) |
| **用户价值** | 解放双手，专注于听讲和思考，自动生成完整的课堂笔记 |
| **复杂度** | 高 |
| **依赖** | 语音识别API/SDK、音频流捕获 |

**用户故事**:
> 作为一名学生，我想要系统自动将课堂视频中的语音转换成文字，以便我能够专心听讲而不用担心遗漏重要内容。

**验收标准**:
- [ ] 支持实时语音转文字，延迟不超过3秒
- [ ] 识别准确率达到90%以上（中文环境）
- [ ] 支持自动分段，按说话人或时间间隔区分段落
- [ ] 支持手动编辑和校对识别结果
- [ ] 识别结果自动关联到对应的课堂笔记

**功能细节**:
- 支持麦克风输入和系统音频捕获两种模式
- 提供识别开关，用户可控制何时开始/停止识别
- 支持标点符号自动添加
- 支持关键词高亮显示

---

##### 功能2: 一键截图保存

| 属性 | 说明 |
|------|------|
| **描述** | 提供快捷键一键截取当前屏幕或指定区域，截图自动插入到课堂笔记中 |
| **优先级** | P0 (核心功能) |
| **用户价值** | 快速保存PPT、板书、代码等视觉内容，与文字笔记形成互补 |
| **复杂度** | 中 |
| **依赖** | 屏幕捕获API |

**用户故事**:
> 作为一名学生，我想要在观看教学视频时快速截图保存重要画面，以便后期复习时能够看到完整的视觉内容。

**验收标准**:
- [ ] 支持全局快捷键触发截图（如Ctrl+Shift+S）
- [ ] 支持全屏截图和区域选择两种模式
- [ ] 截图自动插入到当前课堂笔记的光标位置
- [ ] 截图自动添加时间戳水印
- [ ] 支持对截图进行简单的标注（箭头、框选、文字）
- [ ] 截图以时间戳命名，便于管理

**功能细节**:
- 截图保存格式：PNG
- 截图自动压缩，单个截图不超过500KB
- 支持截图预览和删除
- 支持批量导出截图

## 三、非功能需求

| 指标 | 要求 |
|------|------|
| 启动时间 | ≤ 3秒 |
| 语音识别延迟 | ≤ 3秒 |
| 文档导入时间 | 10MB文档 ≤ 5秒 |

| 指标 | 要求 |

**目标**: 验证核心价值，快速上线获得用户反馈

- ✅ 语音识别转文字（基础版）

### 5.4 技术选型建议

| 模块 | 技术方案 | 说明 |
|------|----------|------|
| 桌面框架 | Electron / Tauri | 跨平台桌面应用框架 |
| 前端框架 | React / Vue | 组件化开发 |
| 语音识别 | 讯飞/百度/系统API | 中文语音识别准确率较高 |
| OCR识别 | 百度OCR / Tesseract | 图片文字提取 |
| 文档解析 | pdf.js + mammoth.js | PDF和DOCX解析 |
| 本地存储 | SQLite / IndexedDB | 本地数据持久化 |
| 富文本编辑 | Quill / Slate | 笔记编辑功能 |

## 六、风险与挑战

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 语音识别准确率不足 | 核心体验受损 | 选择成熟的语音识别服务；提供手动编辑功能 |
| OCR识别效果差 | 图像识别功能不可用 | 提供多OCR引擎选择；支持手动校对 |
| 性能问题（大文档） | 用户体验下降 | 分页加载；后台处理；进度提示 |
| 技术选型不当 | 开发周期延长 | 前期技术验证；选择成熟稳定的方案 |
