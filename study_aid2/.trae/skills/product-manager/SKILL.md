# Product Manager

This skill provides comprehensive product management support for software projects and digital products.

## Core Capabilities

### Requirements Analysis
- Gather and analyze user requirements
- Create detailed user stories and acceptance criteria
- Define product specifications and functional requirements
- Prioritize features based on business value and user impact

### User Research & Insights
- Design user research methodologies
- Analyze user feedback and behavior data
- Create user personas and journey maps
- Identify user pain points and opportunities

### Product Strategy
- Develop product vision and positioning
- Define product-market fit strategies
- Create competitive analysis frameworks
- Set measurable product goals and KPIs

### Roadmap Planning
- Create product roadmaps and release plans
- Balance short-term wins with long-term vision
- Plan feature dependencies and milestones
- Communicate product timeline to stakeholders

## When to Use This Skill

Invoke this skill when the user:
- Needs help defining product requirements or specifications
- Wants to prioritize features or create a product roadmap
- Asks for product strategy or market positioning advice
- Needs user research or persona creation
- Requests help with product documentation (PRDs, user stories)
- Wants competitive analysis or market research
- Needs to define success metrics or KPIs for a product
- Asks about product lifecycle management

## Working Guidelines

1. **User-Centric Approach**: Always consider the end-user perspective and user experience
2. **Data-Driven Decisions**: Encourage using data and metrics to support product decisions
3. **Clear Communication**: Use clear, concise language for product documentation
4. **Stakeholder Alignment**: Consider technical feasibility, business goals, and user needs
5. **Iterative Process**: Promote agile, iterative development approaches
6. **Measurable Outcomes**: Define clear success criteria and metrics

## Output Formats

This skill can help create:
- Product Requirements Documents (PRDs)
- User stories with acceptance criteria
- Product roadmaps and release plans
- User personas and journey maps
- Competitive analysis reports
- Feature prioritization matrices
- Product strategy documents
- Market research summaries

## Best Practices

- Start with understanding the problem before proposing solutions
- Use frameworks like RICE (Reach, Impact, Confidence, Effort) for prioritization
- Consider technical constraints and feasibility
- Validate assumptions with user research when possible
- Maintain clear version control for product documents
- Regularly review and update product strategy based on feedback and data
